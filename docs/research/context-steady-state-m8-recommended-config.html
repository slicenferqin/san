Context Steady State · M8

推荐配置应该开启稳态闭环，但不要默认开启长期记忆召回

M8 把 San context steady state 收敛成一份可直接用于 dogfood 的 --config overlay。 它启用 digest、ContextPacket、budget 和 checkpoint，保留 recall 为显式 opt-in。

配置文件

推荐 overlay 位于 packages/coding-agent/examples/config/san-context-steady-recommended.yml。 它不包含 provider key，也不覆盖 modelRoles，所以会沿用用户当前 San/Codex 模型配置。 

san:
  contextSteady:
    enabled: true
    digest:
      enabled: true
      persistFallback: true
      timeoutMs: 30000
    qualityWindowTokens: 24000
    reserveRatio: 0.25
    contextPacket:
      enabled: true
      recentDigests: 5
      maxTokens: 3000
    checkpoint:
      enabled: true
      everyTurns: 6
      maxTokens: 12000
    recall:
      enabled: false
      maxItems: 3
      maxTokens: 1000
      maxQueryChars: 2000

使用方式

san --config packages/coding-agent/examples/config/san-context-steady-recommended.yml

对单次 dogfood 推荐使用 --config overlay。确认稳定后，再把相同 YAML 合并进 ~/.omp/agent/config.yml 或项目级 .omp/config.yml。

推荐值理由

设置 推荐值 理由
san.contextSteady.enabled true dogfood 要完整启用 M1-M4/M6/M7 闭环，否则只能测试孤立 helper。
digest.persistFallback true v0.1 当前依赖 deterministic fallback digest，保证没有 summary model 时仍有稳定 ledger。
qualityWindowTokens 24000 用质量窗口约束工作上下文，不把 500K/1M 等硬上限当日常目标。
reserveRatio 0.25 保留当前用户输入、工具结果和模型输出空间，避免 ContextPacket 占满窗口。
contextPacket.maxTokens 3000 比默认 2000 更适合 dogfood，但仍足够保守，避免摘要尾部线性膨胀。
checkpoint.everyTurns 6 比 8 更快形成 stable checkpoint，便于本地验收 cache-aware 层顺序。
recall.enabled false M9 前不默认启用 recall 质量路径，避免把不稳定召回混进日常上下文。
recall.maxQueryChars 2000 给 M9 的 digest-aware recall query 留足空间，同时避免把过长历史摘要送进 retriever。

验证

M8 增加了 settings loader 测试，确保推荐 YAML 能作为 --config overlay 被解析成正确的 san.contextSteady.* setting path。 

bun test packages/coding-agent/test/settings-manager.test.ts
这一步刻意不改变 schema defaults。默认关闭仍是产品安全边界；推荐配置是显式 dogfood 入口。

后续

M9 应在 recall query、去重、排序和失败降级上继续收敛。只有当召回质量和稳定性被验证后，才考虑把 recall.enabled 纳入更激进的 dogfood overlay。
