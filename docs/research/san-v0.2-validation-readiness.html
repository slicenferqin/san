用户验收建议

  • 使用 packages/coding-agent/examples/config/san-execution-loop-recommended.yml 显式启用 v0.2。
  • 优先抽样 3 类真实模型任务：小 bug fix、跨文件功能、测试补强。
  • deep 模式只用于架构或发布门禁任务，重点观察 Oracle second opinion 和 Supervisor blocked 质量。
  • 验收时保留 /san-loop status 报告、dogfood script 输出和失败样本，作为 v0.2.1 的改进输入。
默认关闭 v0.2 交付时仍建议默认关闭 execution loop，通过配置或 slash command 显式启用。这样可以交付成熟闭环，同时不改变未选择用户的默认交互路径。
