Context Steady State · M9

Recall 质量优化：更准的 query，更干净的结果，更低的缓存污染

M9 不改变 recall 默认关闭的产品边界。它优化启用 recall 后的 query 构造和结果归一化， 让 retrieved context 继续作为 volatile low-cache layer 留在稳定前缀之后。

代码改动

文件 职责
packages/coding-agent/src/context-steady/recall.ts 新增 San 专用 recall helper：基于当前 prompt 和最近 TurnDigest 构造 query，并对 backend results 去重、清空白、限数。
packages/coding-agent/src/session/agent-session.ts ContextPacket recall 路径改用 helper；只调用 backend search，不调用 save、retain 或 memory prompt 注入。
packages/coding-agent/src/config/settings-schema.ts 新增 san.contextSteady.recall.maxQueryChars，默认 2000。

Query 结构

Recent San turn digests:
- d2: write M7 dogfood verifier | decided t2 | continue t2
- d3: settle M8 recommended config | decided t3 | continue t3

Current prompt:
Improve recall quality

这样做的原因是：单独拿当前 prompt 召回，容易错过上一轮已经明确的项目决策和任务方向； 直接拿完整 transcript 又会过长、噪声大、破坏 retriever 聚焦。最近 digest 是更稳的中间层。

结果归一化

规则 目的
空白 content 直接丢弃 避免无意义 recall item 占用 volatile layer 预算。
优先按 id 去重 同一条 memory 被 backend 重复返回时只保留最高排序的第一条。
无 id 时按 source + normalized content 去重 覆盖纯文本 backend 或旧数据没有 id 的情况。
最多保留 maxItems 召回数量受配置控制，后续仍由 ContextPacket recall token budget 二次裁剪。

边界

M9 仍不把 recall 默认打开。推荐配置保持 recall.enabled: false，因为长期记忆召回质量需要真实 dogfood 后再决定是否升级默认策略。
