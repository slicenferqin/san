Context Steady State · M7

Dogfood 验收要证明 ContextPacket 稳态合同，而不是只证明 helper 能跑

M7 增加了 deterministic dogfood verifier。它不调用外部模型，复用真实 checkpoint、packet builder、 debug report 和 session context rebuild 路径，验证 M1-M6 形成的上下文稳态闭环。

新增产物

文件 职责
packages/coding-agent/src/context-steady/dogfood.ts 生成 10 轮确定性 session entries，构造 checkpoint、ContextPacket、injected custom_message，并返回逐项验收结果。
packages/coding-agent/scripts/context-steady-dogfood.ts 命令行入口；输出每项 dogfood assertion、最终 packet 层顺序、token budget 和 /context packet 同源报告。
packages/coding-agent/test/context-steady/dogfood.test.ts 测试 M7 dogfood verifier 本身，包含成功路径和失败路径，确保它不是只打印固定文案。

验收合同

断言 证明的需求
10/10 turn digests M1 ledger 在多轮里持续存在，后续 packet 有稳定数据源。
至少 1 个 checkpoint M4 stable checkpoint 能覆盖旧 digest，为 cache-aware 稳定前缀提供数据。
1 个 packet + 1 个 injected message M2/M6 debug entry 与实际注入消息可以互相追踪。
stable_checkpoint -> turn_digest_ledger -> retrieved_context 稳定内容在前，append-only digest 居中，volatile recall 最后，符合缓存命中率要求。
packet token estimate 小于等于 token budget M3 预算稳态没有被 dogfood 场景打破。
trim decisions 包含 checkpoint_covered 旧 digest 被 stable checkpoint 覆盖后不会继续重复塞进 digest tail。
active context 隐藏 injected packet，transcript 保留 injected packet resume / replay 时不会无限累计旧 ContextPacket，同时 debug 和 transcript 仍可追踪。

运行方式

bun packages/coding-agent/scripts/context-steady-dogfood.ts
bun test packages/coding-agent/test/context-steady/dogfood.test.ts

verifier 是 deterministic 的，适合本地和 CI 使用。它不依赖 provider key，不启动真实 TUI，不写入用户 session 文件。

边界

M7 不是真实 provider 长会话压测。它验证的是稳态合同和 session replay 路径。真实 CLI dogfood 仍建议在 M8 推荐配置落地后执行。
M7 不评估 recall 语义质量，只验证 recall layer 的位置、预算、只读性质和 debug 可追踪性。召回质量收敛放在 M9。
