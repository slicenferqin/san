覆盖场景

场景 模式 期望 实际 关键事件
Rush pass rush 单轮完成并 finalized。 passed run_created -> plan_created -> assignment_created -> worker_completed -> finalized
Smart retry smart Supervisor 先 needs_fix，Commander 带 review context 重新计划，第二轮 pass。 passed，retry=1 retry_requested 后再次 plan/dispatch/work/finalized。
Deep blocked deep deep policy 先运行 Oracle second opinion，再由 Supervisor 终止为 blocked。 blocked finalized by Oracle 后 review_completed by Supervisor。
Budget exhausted rush 预算耗尽时不再启动下一次 Supervisor agent，直接 blocked。 blocked，reviews=0 blocked with remainingTurns=0
Recovery smart active run resume 时没有 running child process，转为 blocked 并写 recovery event。 blocked recovered
Abort rush operator stop 不删历史，只写 aborted event。 aborted aborted
