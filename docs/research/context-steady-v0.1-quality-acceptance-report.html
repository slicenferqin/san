上下文稳态从「能跑」进入「可验收」

Executive Summary

最终结论：v0.1 质量验收通过

San Context Steady v0.1 在 10 轮验收对话中保持了任务主线、硬约束和质量判断口径。到第 10 轮时， 进入模型的新增 input 只有 598 tokens，核心连续性由 1,612-token ContextPacket 承载；对照组 Codex CLI 在同样 10 轮主题下，第 10 轮 input 达到 198,340 tokens。

这说明 v0.1 的价值不是「多记历史」，而是把多轮历史转译成短、稳、可执行的状态：旧 digest 被 checkpoint 覆盖， 最近 digest tail 保留新变化，当前 prompt 仍保持最高优先级。

行为表现

对照组没有明显跑偏，最终也给出通过，但它通过的是低噪音纯对话场景。风险在于：上下文继续增长后，旧过程、重复解释和一次性估算会越来越容易挤占决策空间。

优点短场景能靠模型长上下文维持主线。
弱点缺少可审计的稳定状态层，无法证明哪些历史被保留、哪些被降级。
失败形态历史化摘要、旧指令残留、重复复述、输入成本线性上升。
没有稳态层时，agent 可以「看起来还记得」，但代价是持续携带越来越大的历史表面；这不是工程稳态，而是窗口预算还没撞墙。

有稳态的 agent 会如何

1. TurnDigest 每轮沉淀用户意图、动作、决策、事实、风险和下一步。
2. Normalize schema drift 不直接失败；进入 normalize + polish，过滤临时记忆。
3. Checkpoint 第 6 轮生成 stable checkpoint，覆盖旧 digest。
4. Packet 下一轮注入 stable checkpoint + 最近 digest tail。
5. Prune provider payload 不再重复发送被 packet 覆盖的原始 transcript。
机制 验收证据 质量含义
TurnDigest ledger 10 / 10 digests；9 个 LLM digest，1 个 fallback digest。 每轮都有结构化状态，不依赖完整历史回放。
Stable checkpoint 1 个 checkpoint 覆盖前 6 个 digest；token estimate 1,205。 旧历史进入高 cache-priority 稳定层，不再反复追加。
Digest tail 最终 packet 只带 2 个 tail digest。 最近变化仍被保留，但不会无限滚动增长。
Packet budget 最终 packet 1,612 / 3,000 tokens。 连续性上下文被硬预算约束，不把模型窗口当垃圾桶。
Transient retry 最终轮 provider stream transient 已补一次 retry 测试。 摘要生成更可靠，失败时仍不会阻塞主 agent turn。

风险与改进项

R1 · LLM digest transient

真实 San dogfood 第 10 轮主回答完成，但 digest LLM 遇到 stream closed transient，走 fallback。已新增一次 retry，并加入测试覆盖。

已缓解
R2 · 摘要历史化

早期 LLM digest 偶尔保留「第 N 轮」叙事。已加入统一 text polish，并覆盖 digest、checkpoint 和 packet 渲染路径。

已优化
R3 · 真实工具噪音仍需压测

本轮报告的 10 轮样本是低工具噪音验收。下一阶段需要加入长日志、diff、文件读取、失败命令和用户改口场景。

M2+ 继续
R4 · Recall 默认关闭

推荐配置保留 recall 为 opt-in，这避免了 v0.1 把未充分验收的长期记忆召回混入稳定上下文。
