M1 唯一目标

每个真实用户轮次最终 settled 后，San 自动生成一个结构化 TurnDigest，并用 sessionManager.appendCustomEntry("san.turn_digest", data) 持久化到当前 session。生成失败不能影响主 agent 结果。
必须做 写入 san.turn_digest custom entry；resume 后可通过 helper 读取。
必须做 有 deterministic fallback digest；LLM 摘要失败、超时、无模型时仍保持 contract。
必须做 Digest 不进入下一轮 LLM context；buildSessionContext() 不能被改变。
必须做 去重：同一 source range / turn id 不能重复写。
不要做 不注入 ContextPacket，不改 prompt 组装策略。
不要做 不写长期 memory，不接 GBrain，不做三角色 loop。

可以按仓库习惯调整文件名，但要保持 digest 逻辑不要散落到巨大的 agent-session.ts 里。agent-session.ts 里只做接线和生命周期保护。

关键实现约束

  1. 接入点必须是 settled turn。 不要写一个普通 subscriber 监听所有 agent_end 就开始写 digest。要接在 AgentSession 内部维护路由确认不会 retry/continue/compaction continuation 后的位置。
  2. Digest job 必须绑定 source range。 至少记录 sessionId、fromEntryId、toEntryId、turnId、createdAt。写入前检查同 range 未写过。
  3. 失败不能阻断主 turn。 LLM digest 失败、JSON parse 失败、timeout、无 summary model，都只能写 fallback 或跳过并 log warning。
  4. Side request 不能触发主 agent loop。 摘要模型调用不能 append message、不能触发工具、不能制造新的 agent_end。
  5. Prompt 必须放静态 markdown。 不要在 TypeScript 里写大段 prompt 字符串；用 import content from "./turn-digest.md" with { type: "text" } 和 Handlebars。
  6. 不要扩大到 M2。 不实现 ContextPacket 注入，不改 append-only context，不接 recall/checkpoint/debug view 的完整体系。

验收标准

  1. 开启 san.contextSteady.enabled 后，真实用户轮次结束会生成一条 customType = "san.turn_digest" 的 entry。
  2. Digest 至少包含 schemaVersion、turnId、sessionId、source、userIntent、actionsTaken、decisions、filesTouched、toolEvidence、nextSteps、fallback。
  3. LLM 摘要失败时，主 agent prompt 不失败，fallback 路径可验证。
  4. Resume 或重新读取 session 后，helper 能列出历史 digest。
  5. buildSessionContext() 不会把 san.turn_digest 注入 LLM messages。
  6. 同一 source range 不重复写 digest。
  7. bun check 和相关 package-local tests 通过。
